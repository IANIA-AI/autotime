## Project in one paragraph

AutoTime needs a governed natural-language-to-SQL assistant for its external Report Writer. The product runs locally in the customer's environment, uses a local model runtime, maps requests into a controlled semantic catalog, generates structured intent, compiles SQL deterministically, applies scope and policy rules, validates the output, and returns copyable SQL to the user.

## Non-negotiable boundaries

The product does not:

- execute SQL in production;
- connect to the production database at runtime;
- call cloud LLM APIs;
- replace AutoTime authorization or runtime security;
- trust free-form SQL directly generated by an LLM.

The preferred path is:

```text
Natural Language
  -> Structured Intent / IR
  -> Semantic Resolution
  -> Deterministic SQL Compiler
  -> Scope Engine
  -> SQL Validators
  -> Governed SQL Output
```

## Current project posture

This is a technical blueprint repository, not an implementation repository yet.

The proposed first delivery target is a **Working Product with engineering evidence**, focused on Labor Charge and Employee report families, local model execution, governed SQL generation, evals, smoke tests and UAT.
